### GitHub Issue

GitHub issue are use to record the issue tracking system that is available in all GitHub repositories.

GitHub issues is used to record

- report bugs
- requesting new features
- documentation
- task
  Github issue can exist without any code.

we need issue to be organized while working in a project.

core components a issue have is :

- Title
- Description
- Issue number - Generated by automatically by GitHub and it is unique.
- Author and timestamp
- Task lists

### `git log`

- It displays the detailed history of commits .

- It shows the hash code, author and timestamp of when it is done, and commit message

### `git log --oneline`

- It shows the compact history of the commits (as oneline)
- It shows short hash and message
